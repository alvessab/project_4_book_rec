Our Project

Why Books?

Have you ever had a hard time deciding what book to read? You ask friends and family but their suggestions ust don't quite hit the mark. We decided to tackle this problem by creating a recommendation program to help you select the book to read.

Our inspiration came by looking at the Anime Recommender that had been done by a previous bootcamp, as well as our own personal love for reading. When looking for data sources, found two other book recommender systems that gave us ideas on how to tackle the problem. These were Toward Data Science, How Did We Build Book Recommender Systems in an Hour and The Clever Programmer, Book Recommendation System.

Our Data

The data set we settled upon is the Goodreads Best Books Ever with Recommendations from Kaggle. You can view the data set here.

The original plan was to use the Goodreads API, but we soon found out that since the site was aquired by Amazon, the API had been shutdown. We decided to go with a dataset that was pulled from the last time the API was active. The dataset consists of 10,000 of the top books from Goodreads. The books that wre pulled were not language specific either, so books that were not in english but were rated high were included in this dataset. Once the data was cleaned we were able to start doing our EDA.

EDA

The most significant part of the initial EDA was removing the unneeded columns of 'recommendations' since we will be creating a program to do that, as well as 'bookISBN' since only 85% of the listed books had them listed. Further EDA was done to allow for NLP to happen. This included sorting the books by their published language and removing the books that were not in English. From here we dropped all duplicates that were found and we had clean CSV that we could do our analysis on.

Recommender Models

The first model that was used is based on natural language processing or NLP. The NLP Recommender uses the cosine similarity of the TF-IDF scores from the bookâ€™s description. It will return the top ten most similar novels to your chosen title.

The second model that we are using is a KNN model. Unlike the NLP model, we are using numeric data to fit the model. Each book is a combination of features that create a data point. The model looks at the distance between each data point and gives recommendations based on the closest data points to the input.
